# Demo and Submission Plan

## Demo objective

Make the project legible to a judge in under 3 minutes.

## Demo structure

### 1. Hook
Prediction markets are powerful, but not always calibrated. MacroFair finds live macro contracts where crowd odds drift from fundamentals.

### 2. Workflow
Show:
- data ingestion
- feature generation
- fair-value estimation
- dislocation scoring
- live app

### 3. Key finding
Show one concrete insight, for example:
- low-liquidity macro contracts are less calibrated
- certain pre-release windows show larger persistent gaps
- cross-platform differences are informative

### 4. Live product
Show:
- ranked screener
- one detail page
- one explanation panel
- methodology page

### 5. Close
Why it matters:
- better market understanding
- transparency
- research value
- reusable live product

## Submission assets checklist

- public repo
- public Zerve project
- working public app or API
- 3-minute demo video
- screenshots
- 300-word summary
- social post
- proof pack snapshot for final milestone
